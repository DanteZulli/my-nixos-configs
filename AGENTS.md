# AGENTS.md

Single-host NixOS flake (`lachata`, user `dante`, x86_64-linux).
Built with the [Den Framework](https://den.denful.dev) (Dendritic Pattern).

System: `nixpkgs-unstable`, home-manager: `master` (semi-rolling, `system.stateVersion` = `"26.05"`).

## Commands

```bash
just rebuild        # alejandra --quiet . + nix run .#lachata -- switch
just update         # nix flake update
just clean          # keep last 10 system + user generations, then nix-collect-garbage --delete-old
just vm             # nix run .#vm-lachata (default host)
just list           # show all recipes
```

`rebuild` runs `alejandra` first — no separate format step.

Syntax check a single file:
```bash
nix-instantiate --eval --strict <file>
```

Regenerate `flake.nix` after input changes:
```bash
nix run .#write-flake
```

## Structure

```
flake.nix              # auto-generated by flake-file — DO NOT EDIT
assets/
  colors.nix           # Qogir palette (imported by foot, mako, sway, waybar)
modules/               # import-tree auto-loads every .nix file (no manual imports needed)
  hosts/
    lachata/
      lachata.nix      # host entity + host/user declaration — includes all host features
      facter.json      # hardware report by nixos-facter (run as root)
  users/
    dante.nix          # user entity — includes all user features
  defaults.nix         # stateVersion, user classes
  dendritic.nix        # flake-file + den flake modules
  nh.nix               # per-host build apps via nh
  vm.nix               # QEMU VM runner (one package per host, e.g. vm-lachata)
  packages.nix         # user packages (stable + unstable)
  opencode.nix         # OpenCode MCP server + agent rules context
  ...
```

## Architecture

- **Module pattern**: `den.aspects.<name>.<class> = { ... }` where `<class>` is `nixos` or `homeManager`.
- **Enabling**: List the aspect in the entity's `includes` (e.g., `den.aspects.boot`). `includes` are dot-access references, not import paths.
- **No feature-gating**: Den's `includes` replaces `options.*.enable` / `lib.mkIf`.
- **Function-call includes** require parens: `(den.batteries.user-shell "bash")`.
- **Host hardware**: [NixOS Facter](https://github.com/nix-community/nixos-facter) replaces `hardware-configuration.nix`. Generate with `sudo nix run nixpkgs#nixos-facter -- -o facter.json` on the target machine. The `hardware.facter` module auto-configures kernel modules, UEFI, graphics, etc. FileSystems still need manual config.
- **Batteries** in `defaults.nix`: `define-user`, `primary-user`, `user-shell`, `hostname` are applied globally via `den.default.includes`.

## OpenCode Permissions (from `modules/opencode.nix`)

| Pattern | Access |
|---------|--------|
| `just *`, `j *`, `sudo *` | **deny** |
| `nix*` (nix, nix-env, nix-store, ...) | **ask** |
| `nixos-rebuild *` | **deny** |
| `gh *`, `rm *` | **ask** |
| `git status*`, `git log*`, `git diff*`, `git show*`, `git remote*`, `git config*`, `git help*` | **allow** |
| `git *` (otros) | **ask** |

## Conventions

- Indent: 2 spaces.
- Functions: `{ den, pkgs, ... }:` — destructure only what's used.
- Naming: kebab-case files, camelCase attrs/vars, dot-separated NixOS options.
- Strings: double quotes normally, `''...''` for multi-line.
- Multi-element lists: one per line. Single-element: inline.

## Git

- Remote: `git@github.com:DanteZulli/my-nixos-configs.git`
- Branch: `main`, `push.autoSetupRemote = true` configured.
- Commits: imperative mood, lowercase, no trailing period.

## Structure

```
flake.nix                                 # entry point
hosts/lachata/                            # NixOS system config
  default.nix                             #   imports modules/nixos + modules/home
  hardware-configuration.nix              #   auto-generated, do not edit
modules/nixos/                            # system-level NixOS modules
  boot, desktop, gaming, hardware,        #   bootloader, X11/Budgie, Steam/Gamescope,
  networking, nix, services, shell,       #   GPU, NetworkManager, nix.conf, PipeWire,
  timezone, users                         #   timezone/locale, users
modules/home/                             # user-level Home Manager modules
  core.nix, packages.nix                  #   username/stateVersion, user packages
  cli/                                    #   git, gh, direnv, opencode
  gui/                                    #   firefox, alacritty, mangohud
  shell/                                  #   zsh, starship
```

## Hard Rules

- **ALWAYS** prefer home-manager (`./modules/home/`) over ad-hoc config when adding user-level settings.
- **NEVER** use `nix-env` (imperative, not reproducible). Use `home.packages` instead.
- **Enable what you import**: Every new feature-gated home-manager module MUST also have its `enable = true;` added to `hosts/lachata/default.nix`. Importing alone does NOT activate the module.

## Notable Patterns

- **`mkHost` helper**: `flake.nix` defines a `mkHost` factory function that wires home-manager, `specialArgs`, and `pkgsUnstable` automatically. Adding a new host is `nixosConfigurations.<name> = mkHost "<name>";`
- **Feature-gated modules**: Every module in `modules/home/cli/`, `modules/home/gui/`, and `modules/home/shell/` defines an `enable` option via `lib.mkEnableOption`. Hosts explicitly enable what they need (e.g. `git.enable = true`).
- **Unstable packages**: `pkgsUnstable` is passed via `home-manager.extraSpecialArgs` in `flake.nix`. Use in `home.packages` via `++ [ pkgsUnstable.pkgname ]` for packages from nixos-unstable (e.g., `opencode` in `modules/home/packages.nix`).
- **`with pkgs;`** only inside `home.packages` lists. Everywhere else, spell out full paths (`lib.lists.optionals`, not `with lib;`).
- **`hardware-configuration.nix`**: auto-generated by `nixos-generate-config`. Never edit directly.

## Code Style

- Indent: 2 spaces.
- Functions: `{ config, lib, pkgs, ... }: { ... }` â€” destructure only what's used.
- Feature-gated module pattern: `let cfg = config.<name>; in { options.<name>.enable = lib.mkEnableOption "..."; config = lib.mkIf cfg.enable { ... }; }`
- One blank line between top-level option blocks; no blank line between an option and its first sub-option.
- Multi-element lists: one per line. Single-element: inline.
- Strings: double quotes normally, `''...''` for multi-line.
- String/naming: kebab-case files, camelCase attrs/vars, dot-separated NixOS options.

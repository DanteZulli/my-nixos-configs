# AGENTS.md

Single-host NixOS flake (`lachata`, user `dante`, x86_64-linux).
Built with the [Den Framework](https://den.denful.dev) (Dendritic Pattern).

System: `nixpkgs-unstable`, home-manager: `master` (semi-rolling, `system.stateVersion` = `"25.11"`).

## Commands

```bash
just rebuild        # format (alejandra) + nix run .#lachata -- switch
just debug          # rebuild with --show-trace --verbose
just update         # nix flake update
just clean          # keep last 10 system + user generations, then nix-collect-garbage --delete-old
just vm             # run a QEMU VM to test the config
just list           # show all available recipes
```

`rebuild`/`debug` run `alejandra --quiet .` first — no separate format step needed.

Regenerate `flake.nix` after input changes:
```bash
nix run .#write-flake
```

Syntax check a single file:
```bash
nix-instantiate --eval --strict <file>
```

Bash alias `j` runs `just` from the nixos dir regardless of cwd:
```bash
alias j='just --justfile ~/.nixos/justfile --working-directory ~/.nixos'
```

## Hard Rules

- **Enable what you include**: importing a module is not enough — each feature aspect needs to be listed in the entity aspect's `includes` (lachata → host features, dante → user features).
- **Host aspects** define `den.aspects.<name>.nixos = { ... }`.
- **User aspects** define `den.aspects.<name>.homeManager = { ... }`.
- **No feature-gating**: den's `includes` mechanism replaces `options.<name>.enable` / `lib.mkIf`.
- **`packages.nix`** is a user aspect included via `den.aspects.dante.includes`.
- **`assets/colors.nix`** provides Nord colors, imported by foot, mako, sway, waybar.

## Notable Patterns

- **`den.default.includes`** in `defaults.nix`: applies batteries globally (define-user, hostname, inputs').
- **`inputs'.nixpkgs-unstable.legacyPackages`**: access unstable packages in home-manager (e.g., `vscode-fhs`).
- **`den.batteries.define-user`**: creates OS user account + home-manager user automatically.
- **`den.batteries.primary-user`**: adds `wheel` + `networkmanager` groups.
- **`den.batteries.user-shell "bash"`**: sets login shell at OS + HM levels.
- **`den.batteries.hostname`**: sets `networking.hostName` from `den.hosts` declaration.
- **Bash** is the default login shell (configured via `den.batteries.user-shell "bash"` + `bash.nix`).
- **OpenCode MCP**: `modules/opencode.nix` configures a local MCP server for `mcp-nixos`.

## Code Style

- Indent: 2 spaces.
- Functions: `{ den, pkgs, ... }:` — destructure only what's used.
- Aspect pattern: `den.aspects.<name>.<class> = { ... }` where `<class>` is `nixos` or `homeManager`.
- Multi-element lists: one per line. Single-element: inline.
- Strings: double quotes normally, `''...''` for multi-line.
- Naming: kebab-case files, camelCase attrs/vars, dot-separated NixOS options.

## Git

- Remote: `git@github.com:DanteZulli/my-nixos-configs.git`
- Branch: `main`, `push.autoSetupRemote = true` configured.
- Commit messages: imperative mood, lowercase, no trailing period.
